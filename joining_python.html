Python Cheat Sheet: Join Types

Choosing the correct merge strategy for imperfect matches

SI ANALYTICS
PAGE 02 / 03
Purpose. Use the how argument to control which unmatched records are retained after a merge.
Core distinction. Left and right joins prioritise one table. Inner joins retain matches. Outer joins retain everything.

A Left join

pd.merge(
  people,
  test_info_xavier,
  on="name",
  how="left"
)
15. “A left join retains every row from the left DataFrame.”
Hint options: place students on the left and use how="left".

B Right join

pd.merge(
  test_info_xavier,
  people,
  on="name",
  how="right"
)
17. “A right join retains every row from the right DataFrame.”
18. “A left join is often easier to reason about because the primary dataset appears first.”

C Inner join

pd.merge(
  people,
  test_info_xavier,
  on="name",
  how="inner"
)
19. “An inner join retains only keys present in both DataFrames.”
Hint options: use on="product" and an inner join.

D Outer join

pd.merge(
  people,
  test_info_xavier,
  on="name",
  how="outer"
)
22. “An outer join retains all keys from both DataFrames.”
Hint options: use an outer join on year.

JoinRows retained
